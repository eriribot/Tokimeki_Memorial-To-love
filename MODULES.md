## 权威状态

- 数值、日期、事件和当前幕以 Zustand + 存档为权威。
- 第一集剧情权威来自真实 `出包王女` 世界书中保持关闭的 `剧情第一集` 条目。代码按 UID
  2/名称只读选择并验证正文，每次生成只注入一份；本地 TXT 只是恢复源，不进入 bundle。
- 当前 preset 实际激活的其他世界书可以补充人物和长期事实，但不能覆盖所选剧情条目的 event/stage 边界。代码不会替缺失或损坏的世界书事实编造剧情答案。
- AI 返回的是正文候选，经本地分页和污染检查后才能进入 GAL。
- 正文长度不是完成条件。模型只能在当前阶段收束后输出精确完成标记；被 token 上限截断、未输出标记或输出错误阶段标记的候选进入
  `parse_error`。该标记能证明响应闭合，不能单独证明所有必要情节语义齐全。
- 新生成正文的标准协议是 `@旁白：叙述` 或
  `@角色名【情绪】：台词`。协议只约束可解析格式，不覆盖当前 preset 的人物塑造、文风、叙事密度或自然篇幅；若 preset 使用
  `<content>`，渲染器只消费其中的 `@`
  正文。旧存档的常见冒号、引号、方括号、Markdown 姓名牌和拆行写法仍可归一化；新生成缺少
  `@`、包含裸引号对白、未知角色或 JSON 时进入 `parse_error`。简短情绪词可自由填写，已知词会映射到立绘表情。
- 超过单页长度的对白会在分页后保留原说话人，不能把后续页静默降级成旁白。
- `TavernHelper.generate()` 返回值只证明生成 API 路线；当前没有创建真实聊天楼层，也没有触发 shujuku/database。
- 保底正文必须显式标记为 `fallback`，不能冒充宿主成功。
