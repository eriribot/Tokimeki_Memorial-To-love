## 目标与权威边界

```text
玩家行动
→ gameStore 结算 AP、时段和当前幕
→ 代码只读关闭的剧情世界书条目，并与运行时事件契约、当前 preset 上下文进入 TavernHelper.generate
→ 返回文本通过阶段完成标记，再经本地抽取、分页和角色/情绪归一化
→ GAL 播放
→ gameStore 推进幕、日期和完成标记
```

模型只负责正文表达。AP、好感、日期、事件触发、资源路径、当前幕和完成状态由代码负责。

## 角色权威

- User 是玩家主角，界面中称“你”；不补未提供的外貌、家庭或过去。
- 夕崎梨子是独立可发展角色，不是 User，不替代浴室初遇、保护菈菈、误告白或婚约剧情位。
- 后续关系由玩家选择和状态发展，第一集 lore 不锁死长期路线。

## 正文协议

输出是可直接播放的纯文本，不是 JSON：

```text
<content>
@旁白：走廊尽头传来急促脚步，你停下动作看向门口。
@菈菈【惊讶】：等等，事情怎么会变成这样？
@你【认真】：先把眼前的问题解决，再谈别的。
</content>
[[STAGE_COMPLETE:事件ID:阶段ID]]
```

- `<content>` 内每个非空行都以 `@` 开头；分页后每段是一页 GAL。
- `@角色名【情绪】：台词` 是带呈现提示的对白，`@旁白：文本` 是叙述；旧存档格式继续兼容。
- 玩家归一为“你”，菈菈归一为“菈菈”。
- 情绪允许简短中文词；已登记语义会映射到现有立绘，资源路径、素材编号和状态差值不由模型输出。
- 完成标记使用实际 URL 编码后的 event/stage
  ID，必须是整份回答最后一个非空行且位于 preset 外围结构之后。缺失或 ID 不匹配时拒绝进入 GAL。
- 规划、规则解释、总结和调试文本不能进入正文渲染。

## 请求上下文

每次生成当前实现包含：

1. 当前 event/stage ID、入口原因、游戏日、时段和地点，以及通用事件完成契约。
2. 代码通过 `getWorldbook('出包王女')` 只读查找
   `UID 2 / 剧情第一集`。条目必须关闭，且正文必须具有第一集根标签和标题标记；否则在调用 AI 前显示错误。
3. 通过校验的完整条目以带 event/stage 元数据的 `<selected_story_lore>`
   块单次注入，`should_scan:false`。本地 TXT 只用于人工恢复条目，不进入运行时 bundle。
4. 当前 preset 自身激活的其他世界书上下文可以补充人物和长期事实。代码不发送 `position:none` 或 `should_scan:true`
   原生扫描键，因此关闭的剧情条目不会再被第二条路径重复激活。
5. 后续阶段额外注入当前存档已接受的前置阶段正文，用于连续性。
6. 同一份完整契约作为 `user_input` 和最后一个 depth-0 system 注入发送，抵抗 preset 中较弱的篇幅目标。
7. `preset_name: 'in_use'`、`max_chat_history: 0`，不读取酒馆公共聊天尾部。

关闭的剧情世界书条目是运行前置条件，不是代码可以替代的事实库。它必须提供与运行时 event/stage 匹配的有序必要情节、可观察结束条件和下一禁止边界；缺失这些资料时，生成会在请求前失败，通用 prompt 和完成标记也不能凭空证明剧情语义完整。

通用完成契约要求：

- 未覆盖全部必达节点和收束条件前，不得关闭正文或输出完成标记。
- 字数、行数和 token 数只控制密度，不是停笔条件。
- 关键节点必须在场景中可观察地发生，不能用摘要或画外跳过代替。
- 完成后立刻停在下一次 AP、下一日期、下一时段或下一阶段之前。
- 节点之间的对白、动作、笑点、镜头、转场与自然篇幅仍由当前 preset 和模型自由创作。

## 宿主接通边界

这条路线可以证明：

- 当前 preset 参与 `TavernHelper.generate`。
- 代码具备从真实世界书只读选择关闭剧情条目并单次注入的路径；当前 preset 仍提供其他实际激活的上下文。
- 本地 TXT 没有进入 bundle，也没有原生扫描键或第二份剧情条目注入。
- 返回正文经过本地协议进入 GAL。

它不能证明：

- 创建了真实 hidden user/assistant 楼层。
- 触发了 `MESSAGE_SENT` 或 `/trigger await=true`。
- shujuku、ACU、qrf_plot 或数据库钩子已经运行。

因此不得称为真同层或正式数据库流程。

## 失败与恢复

```text
idle → loading → ready
                 ↘ error → retry
                         → fallback
```

- 世界书或 API 缺失、剧情条目不存在/重复/误开启/正文不完整、请求失败、缺少阶段完成标记、空正文或污染解析失败进入可见错误态。
- fallback 由玩家明确选择并标记来源，不冒充 Tavern 成功。
- 已接受的幕、消息、当前页和上下文楼层 ID 进入存档；加载态恢复时重新请求当前幕，不重复结算行动。
