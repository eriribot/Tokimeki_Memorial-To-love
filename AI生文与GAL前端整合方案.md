## 目标与权威边界

```text
玩家行动
→ gameStore 结算 AP、时段和当前幕
→ 代码只读验证当前幕需要的关闭世界书条目，并武装下一次原生 World Info 扫描中的条目副本
→ 当前幕开场/结尾提示与最多 6 条已保存主线消息进入 TavernHelper.generate
→ 返回文本经本地抽取、分页和角色/情绪归一化
→ GAL 播放
→ gameStore 推进幕、日期和事件完成记录
```

模型只负责正文表达。AP、好感、日期、事件触发、资源路径、当前幕和完成状态由代码负责。

## 角色权威

- User 是玩家主角，界面中称“你”；不补未提供的外貌、家庭或过去。
- 夕崎梨子是独立可发展角色，不是 User，不替代浴室初遇、保护菈菈、误告白或婚约剧情位。
- 后续关系由玩家选择和状态发展，第一集 lore 不锁死长期路线。

## 正文协议

输出是可直接播放的纯文本，不是 JSON：

```text
<content>
@旁白：走廊尽头传来急促脚步，你停下动作看向门口。
@菈菈【惊讶】：等等，事情怎么会变成这样？
@你【认真】：先把眼前的问题解决，再谈别的。
</content>
```

- `<content>` 内每个非空行都以 `@` 开头；分页后每段是一页 GAL。
- `@角色名【情绪】：台词` 是带呈现提示的对白，`@旁白：文本` 是叙述；旧存档格式继续兼容。
- 玩家归一为“你”，菈菈归一为“菈菈”。
- 情绪允许简短中文词；已登记语义会映射到现有立绘，资源路径、素材编号和状态差值不由模型输出。
- 不要求模型在正文末尾输出阶段完成标记；当前阶段由 AP、日期和 Zustand 中的当前幕选择。
- 规划、规则解释、总结和调试文本不能进入正文渲染。

## 请求上下文

每次生成当前实现包含：

1. 当前幕标题、明确开场、明确结尾、主要角色和 GAL 行协议。
2. 代码通过稳定 UID/名称只读查找 `出包王女` 中关闭的条目：两幕都选择 `剧情第一集`，第二幕额外选择菈菈人物条目。
3. 条目必须保持关闭并通过根标签/内容标记校验；否则在调用 AI 前显示错误。代码只在下一次 `WORLDINFO_ENTRIES_LOADED`
   扫描的内存副本中把所选条目设为常驻、100% 和忽略预算，不修改已保存世界书。
4. 当前 preset 自身激活的其他世界书仍可补充人物和长期事实；一次性钩子结束后立即停止，不污染后续普通扫描。
5. 连续性使用当前存档已保存的主线 user/assistant 消息，过滤当前事件和幕范围后取最后 6 条，写入
   `overrides.chat_history.prompts`，并保留 depth entries。
6. `preset_name: 'in_use'`、`max_chat_history: 6`，不依赖酒馆公共聊天尾部自动拼接。

关闭的剧情世界书条目是运行前置条件，不是代码可以替代的事实库。它必须提供与运行时 event/stage 匹配的有序必要情节、可观察结束条件和下一禁止边界；缺失这些资料时，生成会在请求前失败，通用 prompt 也不能凭空证明剧情语义完整。

当前幕提示要求：

- 未覆盖全部必达节点和收束条件前，不得结束回答。
- 字数、行数和 token 数只控制密度，不是停笔条件。
- 关键节点必须在场景中可观察地发生，不能用摘要或画外跳过代替。
- 写到当前幕明确结尾后立即停笔，不进入下一幕。
- 节点之间的对白、动作、笑点、镜头、转场与自然篇幅仍由当前 preset 和模型自由创作。

## 宿主接通边界

这条路线可以证明：

- 当前 preset 参与 `TavernHelper.generate`。
- 代码具备从真实世界书只读选择关闭剧情/人物条目并武装下一次原生 World
  Info 扫描副本的路径；当前 preset 仍提供其他实际激活的上下文。
- 本地 TXT 没有进入 bundle，已保存条目的关闭状态不会被代码改写。
- 返回正文经过本地协议进入 GAL。

它不能证明：

- 创建了真实 hidden user/assistant 楼层。
- 触发了 `MESSAGE_SENT` 或 `/trigger await=true`。
- shujuku、ACU、qrf_plot 或数据库钩子已经运行。

因此不得称为真同层或正式数据库流程。

## 失败与恢复

```text
idle → loading → ready
                 ↘ error → retry
                         → fallback
```

- 世界书或 API 缺失、剧情条目不存在/重复/误开启/正文不完整、请求失败、空正文或污染解析失败进入可见错误态。
- fallback 由玩家明确选择并标记来源，不冒充 Tavern 成功。
- 已接受的幕、消息、当前页和上下文楼层 ID 进入存档；加载态恢复时重新请求当前幕，不重复结算行动。

## 验证标准

本地自动化应证明：

- 没有独立推进时间按钮；梨子可交谈并增长好感。
- 第一次行动只生成第一幕，入口是 `after_first_action`。
- 第一幕结束后仍是 4 月 7 日、`AP=1`。
- 第二次行动只生成第二幕，入口是 `after_second_action`，并携带第一幕连续性上下文。
- 两次请求使用当前 preset、`max_chat_history: 6`，且没有 `json_schema`；prompt 包含当前幕开场/结尾和 `@旁白` /
  `@角色名【情绪】` 格式，但不要求模型输出 event/stage 完成标记。
- 只有合法 `@` 正文、没有额外完成标记的普通响应可以直接进入 GAL。
- 两幕生成前都只读验证关闭的 `剧情第一集`，第二幕额外验证关闭的菈菈人物条目；本次扫描副本被临时启用，保存状态不变。
- 第二幕 `overrides.chat_history.prompts` 携带已保存的前置主线消息，数量不超过 6 条。
- 条目误开启时在 `generate` 前进入可见错误，AP 保持已结算值；测试不使用固定页数作为完成条件。
- 失败和 fallback 可见，不重复扣点；两幕结束后进入 4 月 8 日并保留唯一事件完成记录。
- 最终 inline HTML 通过 Tavern replacement、legacy entity decode 和脚本语法检查。

真实酒馆人工审查还需证明：`getWorldbook` 实际读到正确的关闭条目、下一次 `WORLDINFO_ENTRIES_LOADED`
只临时启用当前幕所选副本、生成后保存状态仍关闭、第二幕消息连续性正确、无末行完成标记的正文能够分页，以及宿主消息和插件/数据库链没有误触发。
