# Sephie 分层动态立绘（待用户验证）

这个目录包含 Sephie 的轻量分层动态立绘和独立 demo.html。实现方式是 body + Alpha mask + 三帧眼睛图集 + 三帧嘴型图集，不是 Cubism 模型。

本轮只借用 artsource/lala/ 的旧式游戏立绘光线、饱和度、长三分之四身占比和图集格式，不复制菈菈的脸型、绿眼、帽子、翅膀、服装或幼态比例。

## 角色约束

- 构图：接近菈菈素材的长三分之四身，从头发顶部延伸到躯干、双手、胯部和裙摆下段；不是胸像，也不是缩小的头到脚全身图。
- 身体数据：167 cm、50.7 kg、92/58/90 cm。
- 比例换算：胸腰围比约 1.586，臀腰围比约 1.552；画面采用丰满胸围、明显收腰、接近胸围量级的胯臀，同时保持纤细四肢。
- 面纱：只覆盖面部轮廓，从刘海下方经过眼、鼻、嘴到下巴；降低约 45% 的脸部对比度和饱和度。
- 禁止：后置面纱、肩背垂布、婚纱轮廓、蓝色场景背景、菈菈人物特征。

## 运行素材

| 文件 | 尺寸 | 内容 |
| --- | ---: | --- |
| sephie_body.png | 1024×1024 | 中性三分之四身底图 |
| sephie_mask.png | 1024×1024 | 整张立绘 Alpha mask |
| sephie_a_eye.png | 256×512 | 菈菈原格式：睁眼、半闭、闭眼三帧纵排 |
| sephie_a_mouth.png | 256×256 | 菈菈原格式：闭口、中开、大开三帧纵排 |

sephie_body_alpha.png 是透明预览。演示页只读取上表四个运行文件。

## 目录结构

- sources/：本轮生成的 1254×1254 主图和表情源帧。
- variants/：缩放到 1024×1024 的完整表情帧。
- drafts/previous-bust-demo/：上一版胸像运行素材和源图。
- 其他 drafts/：全身比例、后置面纱、婚纱式面纱及蓝底等被否决稿。

## 当前窗口配置

demo.js 顶部集中保存坐标：

~~~js
regions: {
  eyes: { x: 400, y: 90, width: 225, height: 145 },
  mouth: { x: 420, y: 185, width: 185, height: 105 },
}
~~~

眼图集和嘴图集采用菈菈素材的 256×512 / 256×256 文件尺寸，运行时分别缩放进上述 Sephie 专属窗口。

## 打开 demo

~~~powershell
cd src/Tokimeki_Memorial-To-love/artsource/sephie
python -m http.server 4173
~~~

然后打开：

~~~text
http://localhost:4173/demo.html
~~~

诊断工具栏可分别关闭 `显示 eyes` 与 `显示 mouth`，用于单独检查 body 底图和两块动态贴片；点击“重置演示”会恢复两层显示。

## 待用户验证

本轮按要求没有启动浏览器或替用户验收。建议自行检查：

1. 素材列表是否显示 4 / 4，尺寸是否为 1024×1024、1024×1024、256×512、256×256。
2. 睁眼、半闭、闭眼三帧切换时，刘海、面纱边缘和额饰是否稳定。
3. 闭口、中开、大开三帧切换时，下巴、面纱和项圈是否稳定。
4. 面纱下的眼鼻嘴是否保持明显雾化，而不是突然恢复清晰。
5. 浅色、深色和粉色检查底下是否有背景残边。
6. 立绘占比是否与 lala/004_03_03_a #2247.png 同属长三分之四身，而非胸像或远距离全身。
